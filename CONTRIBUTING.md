# Contributing to AutoJoin for SteamGifts

Thank you for your interest in contributing! This document outlines the process for contributing to this project.

## Getting Started

1. Fork the repository on GitHub
2. Clone your fork locally
3. Create a feature branch from `master`
4. Make your changes
5. Test your changes
6. Submit a pull request

## Development Workflow

### Branching Strategy

- `master` - Main development branch
- Feature branches - Create from `master` for new features or fixes
- Use descriptive branch names: `feature/add-new-feature` or `fix/bug-description`

### Pull Requests

1. Ensure your branch is up to date with `master`
2. Write clear commit messages
3. Provide a detailed PR description explaining:
   - What changes were made
   - Why the changes are needed
   - How to test the changes
4. Reference any related issues

### Code Style

- Follow existing code patterns in the project
- Use meaningful variable and function names
- Add comments for complex logic
- Keep functions small and focused

### Testing

- Test your changes in multiple browsers if applicable
- Ensure the extension still works as expected
- Check for any console errors

## Project Structure

- `manifest.json` - Extension manifest
- `js/` - Core JavaScript files
- `core/` - Modular components
- `html/` - HTML pages
- `css/` - Stylesheets
- `media/` - Images and assets

## Questions?

If you have questions about contributing, feel free to open an issue or start a discussion in the repository.
